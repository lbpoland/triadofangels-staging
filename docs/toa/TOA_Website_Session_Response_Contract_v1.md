# TOA Website Session Response Contract (Paste into future project work if needed)

Use this response template for every implementation session.

## 1) Patch Summary
- What was changed
- Why it was changed
- What user-visible impact to expect

## 2) File Manifest (Exact Paths)
List every changed/new file and mark:
- Added
- Updated
- Not included yet (if still in progress)

## 3) Downloads
- One authoritative ZIP (contains only manifest files)
- Individual file downloads (recommended if practical)

## 4) Website Work Checklist
### DONE
- ...
### NEXT
#### P0
- ...
#### P1
- ...
#### P2
- ...
#### P3
- ...
### Risks / Notes
- ...

## 5) QA Checklist (PASS / FAIL / NOT RUN)
- Dev-check strict
- Lighthouse mobile
- Lighthouse desktop
- Keyboard nav
- Mobile menu
- Dropdowns
- Theme toggle
- Reduced motion
- Forced-colors sanity check
- Canonicals/meta/OG/Twitter
- JSON-LD
- Broken links
- Console errors

## 6) Continuation Prompt (Copy/Paste)
Must include:
- authority order
- current status summary
- files modified this session
- next priority tasks
- output rules (ZIP + manifest + checklists)
